# 指南处理工作总结

## 问题描述

原始的指南 JSON 文件存在以下问题：
1. 章节标题包含大量页眉、页脚等无关信息
2. 章节划分过细（507 个章节），不利于检索和使用
3. 内容结构混乱，缺乏逻辑性

## 解决方案

采用**规则优化 + LLM 智能优化**相结合的方法：

### 1. PDF 文本提取
使用 `pdfplumber` 库从 4 个 PDF 指南文件中提取文本：
- 乳腺癌诊疗指南（2022 版）.pdf
- 中国乳腺癌随诊随访与健康管理指南 (2022 版).pdf
- 中国乳腺癌内分泌治疗专家共识（2015）.pdf
- 乳腺癌随访及伴随疾病全方位管理指南.pdf

### 2. 初步结构化
创建 `scripts/process_guidelines_llm.py` 脚本：
- 从 PDF 提取文本
- 使用简单规则初步划分章节
- 支持 LLM 智能优化（需要配置 API Key）

### 3. 规则优化
创建 `scripts/optimize_guidelines.py` 脚本：
- 清理章节标题中的无关信息
- 合并内容过短的相邻章节
- 过滤掉内容质量差的章节
- 自动提取版本号和发布机构信息

### 4. LLM 智能优化（可选）
创建 `scripts/llm_optimize_guidelines.py` 脚本：
- 使用 LLM 理解指南内容
- 智能重新划分章节结构
- 生成清晰准确的章节标题
- 合并相关内容，去除无关信息

## 处理结果

### 优化效果

**优化前问题：**
- 章节标题如："2012 年数据报道，全世界每年有超过 167 万女性 临床操作中，难免存在差距和偏移。"
- 内容混杂页眉页脚："guide.medlive.cn 《中国癌症杂志》2015 年第 25 卷第 9 期"

**优化后改进：**
- 章节标题更简洁准确
- 内容更连贯完整
- 去除了页眉页脚等无关信息
- 章节数量合理（16-64 个），便于检索和使用

## 使用方法

### 使用指南处理器

```python
from src.data_processing.guideline_processor import GuidelineProcessor

# 初始化处理器（自动加载优化后的指南）
processor = GuidelineProcessor()

# 搜索相关内容
sections = processor.search_sections("内分泌治疗")
for section in sections:
    print(f"来源：{section['guideline_name']}")
    print(f"章节：{section['section_name']}")
    print(f"内容：{section['content'][:200]}...")

# 获取治疗推荐
recommendations = processor.get_treatment_recommendations("乳腺癌")
```

### 使用效度分析器

```python
from src.meta_evaluation.validity_analyzer import ValidityAnalyzer

# 初始化分析器（自动加载优化后的指南）
analyzer = ValidityAnalyzer()

# 搜索指南内容
results = analyzer.search_guideline_content("随访")
```
